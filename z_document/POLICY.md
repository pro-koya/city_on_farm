あなたは、日本のWebサービスにおける
個人情報保護・セキュリティ・UI/UXに精通した
シニアWebエンジニア兼テクニカルライターです。

以下の条件をすべて満たす
「プライバシーポリシーページ」を作成してください。

# 【目的】
- Stripe 本番審査・各種外部サービス審査に耐えられる
- 日本の個人情報保護法に配慮した内容になっている
- ユーザーに「安心して使えるサービス」という印象を与える
- セッツマルシェの他ページとデザイン・UXを統一する

# 【前提条件】
- サービス名：セッツマルシェ
- 運営形態：個人開発のマーケットプレイス型EC
- 取扱商品：農作物
- 利用者：
  - 一般消費者
  - 飲食店
  - 出品者（農家）
- 決済：
  - Stripe クレジットカード決済
  - 代金引換
- 外部サービス：
  - Stripe
  - Google（ログイン / OAuth / reCAPTCHA を使用する可能性あり）
- ユーザー登録あり
- 配送のため住所・氏名・電話番号を取得

# 【実装要件】
## 1. ページ仕様
- URL想定： /privacy
- EJSテンプレートで実装
- header / footer / ナビゲーション / フォントは既存ページと完全に共通
- SEOを意識した title / meta description を設定
- モバイル・PC両対応

## 2. デザイン要件（重要）
- 利用規約・特商法ページと同等のデザイン品質
- 長文でも「読みやすい」「威圧感がない」構成
- セクション分割＋余白を意識
- 見出し階層を明確に（h1 / h2 / h3）
- dl / ul / p を適切に使い分ける

## 3. 記載すべき内容（必須）
以下を **日本の個人情報保護法に配慮した自然な日本語** で記載してください。

### 基本情報
- プライバシーポリシーの目的
- 運営者情報（事業者名）
  - ※住所・電話番号は「請求があった場合に遅滞なく開示」表現でOK

### 取得する個人情報
- 氏名
- メールアドレス
- 電話番号
- 配送先住所
- 決済に関する情報（※カード番号は保持しないことを明記）
- ログ情報（IP、ブラウザ情報等）

### 利用目的
- 商品の配送
- 決済処理
- 本人確認
- お問い合わせ対応
- 不正防止
- サービス改善

### 第三者提供・外部委託
- Stripe 等の決済代行業者
- 配送業者
- 法令に基づく場合

### Cookie・アクセス解析
- Cookieの使用目的
- セッション管理
- 不正防止

### 個人情報の管理
- 適切な安全管理措置
- アクセス制限
- 不正アクセス防止

### 開示・訂正・削除
- ユーザーからの請求方法
- 連絡先

### ポリシーの変更
- 内容変更時の告知方法

# 【出力形式】
以下を必ず出力してください。

① EJSテンプレート（views/pages/privacy.ejs 想定）
② ページ専用CSS（styles/privacy.css）
③ SEO用 meta 情報（title / description）
④ フッター用リンク文言（例：「プライバシーポリシー」）
⑤ Stripe審査提出時に使えるページ説明文

# 【重要な制約】
- 「City on Farm」という名称は使用しない
- コピペ感のある汎用テンプレは禁止
- 不安を煽る表現は禁止
- 法的に怪しい断定表現は禁止
- 「準拠しています」と断言しすぎない表現を使う
