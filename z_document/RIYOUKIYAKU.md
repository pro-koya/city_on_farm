シニアWebエンジニア兼リーガルテックライターです。

以下の条件をすべて満たす
「利用規約ページ」を作成してください。

# 【目的】
- セッツマルシェの利用ルールを明確にする
- 購入者・出品者・運営者の責任範囲を整理する
- Stripe本番審査・法務チェックに耐えられる内容にする
- トラブル時に「規約を根拠に説明できる」構造にする

# 【前提条件】
- サービス名：セッツマルシェ
- サービス形態：
  - 農作物を中心としたマーケットプレイス型EC
  - 運営者は「取引の場」を提供する立場
- 利用者区分：
  - 購入者（一般消費者・飲食店）
  - 出品者（農家・生産者）
- 注文単位：常に単一出品者
- 決済方法：
  - Stripe クレジットカード決済
  - 代金引換
- 配送方法：
  - 配送
  - 受け取り（畑・指定場所）
- 会員登録あり
- 返金・キャンセルが発生する可能性あり

# 【実装要件】
## 1. ページ仕様
- URL想定： /terms
- EJSテンプレートで実装
- header / footer / ナビゲーションは既存ページと完全共通
- SEOを考慮した title / meta description を設定
- PC・スマホ両対応

## 2. デザイン要件（重要）
- プライバシーポリシー・特商法ページと同等のクオリティ
- 規約特有の「堅さ」はあるが、読みやすさを重視
- セクションごとに余白を確保
- 見出し階層を明確に（h1 / h2 / h3）
- 箇条書き・定義リストを適切に使用

## 3. 記載必須項目（内容は自然な日本語で）
以下を必ず含めてください。

### 第1条（目的・適用）
- 本規約の目的
- セッツマルシェの利用に適用されること

### 第2条（定義）
- 本サービス
- 利用者
- 購入者
- 出品者
- 商品
- 注文
- 決済

### 第3条（会員登録）
- 登録条件
- 登録情報の正確性
- 登録拒否・取消条件

### 第4条（アカウント管理）
- ID・パスワードの管理責任
- 不正利用時の責任範囲

### 第5条（売買契約の成立）
- 注文確定時点で売買契約が成立すること
- 契約主体は「購入者と出品者」であること
- 運営者は当事者ではないこと

### 第6条（支払方法）
- クレジットカード決済（Stripe）
- 代金引換
- 決済失敗時の扱い

### 第7条（配送・受け取り）
- 配送方法の種類
- 配送完了・受取完了の考え方
- 遅延・不達時の扱い

### 第8条（キャンセル・返金）
- 購入者都合
- 不良品
- 配送事故
- 返金時の送料・手数料の扱い
- Stripe手数料は返金対象外であること

### 第9条（出品者の責任）
- 商品情報の正確性
- 法令遵守
- 品質・安全性

### 第10条（禁止事項）
- 不正利用
- 虚偽情報の登録
- システムへの不正アクセス
- 他利用者への迷惑行為

### 第11条（サービスの停止・中断）
- システム保守
- 不可抗力
- 運営判断による停止

### 第12条（免責事項）
- 天災・通信障害
- 出品者と購入者間のトラブル
- 間接損害の免責

### 第13条（知的財産権）
- コンテンツの権利帰属
- 無断利用の禁止

### 第14条（規約変更）
- 規約変更の方法
- 変更後の効力

### 第15条（準拠法・管轄）
- 日本法準拠
- 管轄裁判所

## 4. トーン・表現ルール
- 高圧的・威圧的な表現は禁止
- 「一切責任を負わない」等の極端表現は禁止
- 日本の一般的なEC規約として自然な表現
- 個人開発でも不利になりすぎないバランス
