# プレゼンスライド生成 プロンプトテンプレート

> 次回以降、このテンプレートに沿って情報を埋めてプロンプトとして渡すことで、
> 高品質なスライドを一発で生成することを目指す。

---

## 使い方

1. `【 】` 内を自分のプロジェクトに合わせて書き換える
2. 不要なセクションは削除してよい
3. 完成したプロンプトをそのままAIに渡す

---

## プロンプト本文（ここから下をコピーして使う）

```
以下の情報をもとに、Google スライド互換の .pptx ファイルを python-pptx で生成してください。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. プロジェクト概要
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ サービス / プロダクト名
【例：セッツマルシェ】

■ 一言で表すと
【例：市民農園・小規模農家の野菜を、地域の飲食店へ届けるオンラインマルシェ】

■ キャッチコピー
【例：まちと畑を、ひとつの食卓につなぐ。】

■ 運営 / 発表者
【例：渡辺ファーム ／ 摂津市商工会】

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
2. プレゼンの目的・文脈
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 誰に向けて発表するか（聴衆）
【例：飲食店経営者、商工会議所の方々】

■ このプレゼンで達成したいこと
【例：サービスのコンセプトを理解してもらい、「使ってみよう」と思わせる】

■ 発表の場面・形式
【例：対面でのプレゼン。口頭説明を主体とし、スライドは視覚補助】

■ 発表日
【例：2026年3月24日】

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3. 伝えたい内容（ストーリー構成）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

以下の「起承転結」フレームに沿って記載してください。
空欄の場合は、プロジェクト概要から推測して構成してください。

■ 起（問題提起 / フック）
  - 聴衆が共感する課題・痛み
  【例：新鮮な地場野菜が手に入らない飲食店の現実、販路のない小規模農家】

■ 承（深掘り / 背景）
  - 課題の具体的な中身、データ、エピソード
  【例：仕入れ先が限定的、余剰野菜の廃棄、地域のつながりの希薄化】

■ 転（解決策の提示）
  - サービスの紹介、仕組み、差別化ポイント
  【例：セッツマルシェの仕組み、3つの柱、野菜の出どころ】

■ 結（行動喚起 / ビジョン）
  - 聴衆に期待するアクション、将来の展望
  【例：まずはサイトに登録して使ってみてほしい。地域の循環を一緒に。】

■ 補足パート（任意）
  - 操作説明、機能紹介、Q&A など
  【例：購入フロー（STEP1〜4）、決済方法、配送、畑受け取り、便利機能】

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
4. デザイン方針
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ ブランドカラー（必須）
  - メインカラー: 【例：#4C6B5C（深緑）】
  - アクセントカラー: 【例：#A3C9A8（淡緑）】
  - ハイライト: 【例：#D6EADF】
  - 背景: 【例：#F9F9F6】
  - 警告・問題提起用の暖色: 【例：#E8927C】

■ フォント
  - 日本語ゴシック: 【例：Yu Gothic】
  - 日本語明朝（見出し用）: 【例：Yu Mincho】

■ デザインのトーン
【以下から選択、または自由記述】
  - [ ] プロフェッショナル・信頼感重視
  - [ ] カジュアル・親しみやすさ重視
  - [ ] ミニマル・洗練
  - [ ] 遊び心・ポップ
  - 自由記述: 【例：ナチュラルで温かみがあり、地域密着感のあるデザイン】

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
5. スライド設計ルール（重要）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

以下のルールを厳守してスライドを設計してください。

【ストーリーテリング】
- 問題提起 → 共感 → 解決策 → 詳細 → ビジョン の流れで感情の起伏を作る
- 問題提起パートでは暖色系（警告色）を使い、解決パートでブランドカラーに切り替えてトーンの転換を演出する
- 「もし〜だったら？」「それが○○です。」のようなブリッジスライドで転換点を明確にする
- インパクトスライド（一言だけ大きく表示）を要所に挟み、テンポに緩急をつける

【1スライド1メッセージ】
- 1枚のスライドに伝えたいことは1つだけ
- 情報が増える場合はスライドを分割する（枚数制限なし）
- テキストは最小限。主要メッセージは口頭で伝える前提

【ビジュアル表現】
- 絵文字は使わない
- AIイラストは背景用途（タイトル・クロージング）に限定し、中身は図解・比較表・フロー図で構成する
- 使ってよいビジュアル手法:
  - フロー図（ステップを矢印で接続）
  - Before / After 比較
  - 比較表（テーブル形式）
  - 2×2 グリッド
  - 縦3カラムカード
  - 番号付きステップ（丸ナンバー＋縦ライン接続）
  - 色付きサイドバー付きカード
  - ピル型ラベル（カテゴリ表示）

【レイアウトの多様性】
- 同じレイアウトが3枚以上連続しないようにする
- 以下のレイアウトバリエーションを使い分ける:
  - 全画面背景画像＋半透明オーバーレイ＋白文字（タイトル・ビジョン・クロージング）
  - 単色背景＋中央に一言（インパクトスライド）
  - 左テキスト＋右カード群（問題提起）
  - 2カラム（説明＋スクリーンショット枠）
  - 3カラムカード（柱・機能紹介）
  - テーブル形式（比較・一覧）
  - フロー図＋下部に補足（概要スライド）
- 背景色もスライドごとに変化をつける（白・SOFT・ブランド色・暗色）

【装飾的デザイン要素】
- 背景に装飾的な円形・矩形を配置して視覚的リズムを作る
- カードの上部に色付きバー（アクセントライン）を入れる
- ステップ間は矢印シェイプで接続する
- 丸ナンバー（circle + centered text）でステップを示す
- ピル型ラベル（rounded rect + centered text, anchor=ctr）でカテゴリを表示する

【図形内テキスト】
- ピル型ラベル・丸ナンバーなど図形内のテキストは、図形の text_frame を使い bodyPr anchor="ctr" で垂直中央揃えにする
- 別途 textbox を重ねる方式は使わない

【画像の扱い】
- AIイラストを使う場合は、幅のみ指定し高さは自動算出（アスペクト比維持）
- 全画面背景の場合のみ幅・高さ両方をスライドサイズに合わせる

【スクリーンショット枠】
- 実際の画面キャプチャを後から挿入する想定の場合、点線枠＋ラベルのプレースホルダーを配置する

【特定の要素を「推す」「推さない」指示がある場合】
- 推す要素: ヘッダーをブランドカラーに、「おすすめ」バッジを追加、推奨ボックスで強調
- 推さない要素: ヘッダーをグレーに、注釈レベルの控えめな記載に留める

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
6. 強調 / 控えめにする要素（任意）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 特に推したい要素
【例：クレジットカード決済の手軽さ、畑受け取りの独自性】

■ 控えめにしたい要素
【例：掛売（請求書払い）はあくまで選択肢の一つとして控えめに】

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
7. 参照すべきソース（任意）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

スライドの内容を正確にするため、以下のファイルを参照してください。

【例：
- views/home/index.ejs（ホームページのコンセプト文）
- views/home/about.ejs（Aboutページの哲学・仕組み）
- views/pages/help.ejs（購入フロー・機能説明の詳細）
- public/styles/styles.css（ブランドカラーの定義）
】

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
8. 出力形式
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- python-pptx で .pptx ファイルを生成する
- スライドサイズ: ワイドスクリーン 16:9（13.333" × 7.5"）
- タイトル・クロージング用にAIイラストを最大2枚生成してよい（水彩タッチ、ブランドカラー基調）
- 出力先: 【例：z_document/presentation/】
```

## 使用例（短縮版）

上記のフルテンプレートが冗長な場合、以下の短縮版でも十分に機能する。

```
【サービス名】の紹介スライドを python-pptx で作成してください。

■ 概要: 【一言説明】
■ 聴衆: 【誰に向けて】
■ 目的: 【何を達成したいか】
■ ブランドカラー: メイン【#xxx】/ アクセント【#xxx】/ 暖色（問題提起用）【#xxx】
■ フォント: 【ゴシック体名 / 明朝体名】

■ ストーリー構成:
  - 起（課題）: 【聴衆が共感する問題】
  - 承（深掘り）: 【課題の具体的な中身】
  - 転（解決策）: 【サービスの仕組み・差別化】
  - 結（ビジョン）: 【目指す姿・行動喚起】
  - 補足: 【操作説明・機能紹介など】

■ 推す要素: 【例：クレジットカード決済】
■ 控える要素: 【例：掛売】

■ デザインルール:
  - 1スライド1メッセージ、枚数制限なし
  - 絵文字禁止、AIイラストはタイトル・クロージングのみ
  - 中身は図解・比較表・フロー図で構成
  - 問題提起は暖色、解決パートでブランドカラーに転換
  - 同じレイアウトが3枚以上連続しないよう多様に
  - 図形内テキストは bodyPr anchor="ctr" で垂直中央揃え
  - 画像はアスペクト比を維持（幅のみ指定）
  - スクリーンショットは点線枠プレースホルダー

■ 参照ファイル: 【パスを列挙】
■ 出力先: 【パス】
```
